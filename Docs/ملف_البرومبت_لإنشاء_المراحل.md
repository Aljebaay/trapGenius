# الخطوة التالية لتفعيل نظام توليد المراحل (Devilbait Level Generator)

## 5️⃣ الخطوة التالية (ضرورية)

قبل أن تستخدم النظام لأول مرة، يجب أن تجعل Cursor يقوم ببناء ملف الـ log الخاص بالمراحل.

اكتب له الأمر التالي:

Initialize LEVEL_PLAN_LOG.json from LEVEL_001–LEVEL_030 and add log-sync metadata to each level file if missing.

سيقوم Cursor بالخطوات التالية:

1. قراءة جميع ملفات المراحل من LEVEL_001 إلى LEVEL_030  
2. إنشاء ملف LEVEL_PLAN_LOG.json  
3. إضافة metadata إلى ملفات المراحل إذا كانت غير موجودة  
4. مزامنة النظام بالكامل بين ملفات المراحل وملف الـ log  

---

## 6️⃣ بعد ذلك سيصبح استخدام النظام بسيطاً جداً

بعد التهيئة الأولى، كل ما ستحتاج لكتابته مستقبلاً هو أوامر بسيطة مثل:

### إنشاء مرحلة جديدة
Generate next level plan

### تعديل مرحلة موجودة
Update LEVEL_012: make the trap trigger slightly later

### استبدال مرحلة بالكامل
Replace LEVEL_018 with a rhythm-based trap instead of attempt-based

---

## 7️⃣ ملاحظة مهمة جداً بخصوص مشروعك

النظام الذي بنيته الآن داخل Devilbait هو في الحقيقة:

AI Assisted Level Design Pipeline

أي نظام تصميم مراحل مدعوم بالذكاء الاصطناعي.

وهذا النوع من الأنظمة نادر حتى في ألعاب شهيرة مثل:

- Super Meat Boy
- Geometry Dash

معظم هذه الألعاب تقوم ببناء المراحل يدوياً بالكامل بدون أي نظام توليد أو إدارة ذكي للمراحل.
